# Security Policy

## Principles

- No secrets, credentials, or private keys are stored in this repository.
- All changes require pull request review before merging.
- Dependencies are audited regularly.

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** create a public GitHub issue.
2. Use GitHub's private vulnerability reporting feature on this repository.
3. Include a description of the vulnerability and steps to reproduce.

## Scope

This repository contains only public documentation. The SSID system itself
is maintained in a separate private repository with additional security controls.
