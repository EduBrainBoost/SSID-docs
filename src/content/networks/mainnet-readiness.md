## Evidence Standards

### Required Evidence

All mainnet readiness claims require:

1. **Publicly verifiable evidence**
   - Links to audit reports
   - Test result summaries
   - Security certificates

2. **Timestamped documentation**
   - When was the check completed?
   - Who performed the check?
   - Where is the evidence stored?

3. **Independent verification**
   - Third-party audits
   - Community review
   - Public testnets

### Evidence Format

```yaml
requirement: "Smart Contract Audit"
status: "completed"
evidence:
  type: "third_party_audit"
  auditor: "Security Firm Name"
  date: "2024-04-12"
  report_url: "https://..."
  summary: "No critical issues found"
verified_by:
  - "@auditor_handle"
  - "@ssid_team"
```

## Mainnet Claim Guard

:::caution
**Strict Policy: No Mainnet Claims Without Evidence**

Any documentation, code, or communication claiming:
- "Mainnet is live"
- "Production ready"
- "Active mainnet"

**MUST** include:
- Block number of first transaction
- Contract addresses (verified)
- Evidence links
- Snapshot date

**Without evidence, these claims are invalid and blocked.**
:::
