## Current Phase: Foundation

The SSID project is in its **foundation phase**, establishing the core infrastructure, governance model, and compliance framework.

<CardGrid>
  <Card title="24x16 Matrix" icon="approve-check">
    Structure skeleton complete. All 24 roots and 16 shards are defined with deterministic SoT references.
  </Card>
  <Card title="EMS (External Management System)" icon="approve-check">
    CLI-first control plane (`ssidctl`) operational. 3-worktree model, guard/gate matrix, and evidence store active.
  </Card>
  <Card title="PR-Only Governance" icon="approve-check">
    Enforced across all repositories. Branch protection, CI gates, and WORM evidence logging in place.
  </Card>
  <Card title="Public Documentation" icon="approve-check">
    Starlight-based docs site with deterministic ingest from open-core. No LLM-generated content.
  </Card>
</CardGrid>

## What Will NOT Be on This Roadmap

- Internal security audit timelines
- Token launch dates or pricing
- Infrastructure provider details
- Anything requiring private repository access
