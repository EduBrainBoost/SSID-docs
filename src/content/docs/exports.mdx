## Export Pipeline

The public export process is fully deterministic and enforces multiple safety layers:

1. **Policy glob filter** removes internal paths (audit logs, quarantine, registry internals)
2. **Secret scanner** rejects files containing API keys, private keys, passwords, or tokens
3. **Binary filter** blocks non-allowlisted binary file types
4. **Size cap** rejects files exceeding the configured maximum
5. **Manifest generation** records SHA-256 hashes for every exported file

<Aside type="caution">
The weekly PR export is **approval-gated**. It cannot be activated without a recorded approval in the EMS Approval Ledger. Even when active, it creates a pull request for review — it never auto-merges.
</Aside>

## Integrity

Each export run produces a manifest containing SHA-256 hashes per file. The manifest itself is stored in the EMS runs directory (outside any git repository) and is available for audit verification.

No raw file content, internal identifiers, or absolute paths appear in export manifests or on this page.
