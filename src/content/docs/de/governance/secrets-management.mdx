SSID erzwingt eine strikte No-Secrets-in-Repo Policy durch mehrere Detektions- und Präventionsebenen.

## Aktuelle Implementierung

### Secret Scanning (CI)

| Repo | Tool | Config |
|------|------|--------|
| SSID | Gitleaks v8.21.2 | `23_compliance/gitleaks.toml` |
| SSID-open-core | Grep-basierte Muster | In `gates.yml` secret_scan Job |
| SSID-docs | Grep-basierte Muster | In `docs_ci.yml` secret-scan Job |

Erkannte Muster: `PRIVATE` + `.KEY`, `sk_` + `live_`, `ghp_*`, `gho_*`, `password=`, `api_key=`, `-----BEGIN`, JWT Tokens, Base64 Secrets.

### PII Redaction (EMS Sanitizer)

Das `sanitizer.py` Modul redactiert empfindliche Daten vor Persistierung:

| Muster | Ersetzung |
|--------|-----------|
| Windows User Pfade | `<WIN_USER_PATH_REDACTED>` |
| Unix Home Pfade | `<UNIX_HOME_PATH_REDACTED>` |
| API Keys / Bearer Tokens | `<API_KEY_REDACTED>` |
| Lange Hex Strings (40+ Chars) | `<HEX_REDACTED>` |
| Email Adressen | `<EMAIL_REDACTED>` |
| Environment Variablen mit Werten | `<ENV_REDACTED>` |

### Evidence Policy

- **Hash-Only**: Evidence speichert SHA-256 Hashes, nie rohe Inhalte
- **Redaction Enforcement**: Findings ohne `redacted=true` werden von EvidenceStore abgelehnt
- **Keine rohen Prompts/Responses**: LLM Interaktionen werden nur als Hashes gespeichert

### Pre-Flight Prompt Redaction

Das AI Gateway redactiert Secrets/PII von Prompts **bevor** es zur LLM sendet:
1. Prompt wird durch `contains_secrets()` gescannt
2. Wenn Secrets gefunden: `sanitize_text()` ersetzt sie mit Placeholders
3. Original Prompt Hash protokolliert für Audit (nur Hash, kein roher Text)

## Geplante Verbesserungen

### HashiCorp Vault Transit

- **Encryption as a Service**: Daten verschlüsselt ohne Keys in Applikation zu speichern
- **Signing/Hashing/HMAC**: Integrierte kryptographische Operationen
- **Key Rotation**: Automatisierte, auditable Key Lifecycle

### Cloud KMS/HSM

- **AWS KMS** oder **Azure Key Vault** für Produktions Key Management
- **HSM-gestitzte Keys** für höchste Sicherheit (FIPS 140-2 Level 3)
- **Audit Trail**: Alle Key Operationen protokolliert via CloudTrail / Azure Monitor

### Encryption at Rest

Aktuell: Plaintext JSON/JSONL auf Filesystem (akzeptabel für lokale Entwicklung).
Ziel: Verschlüsselte Volumes oder Vault Transit für alle Evidence und State Daten.
