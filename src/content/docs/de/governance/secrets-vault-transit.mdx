## Übersicht

[Vault Transit](https://developer.hashicorp.com/vault/docs/secrets/transit) bietet Encryption as a Service — Applikationen sehen niemals rohes Key Material.

## Architektur

```
Application → Vault Transit API → HSM-gestitzte Keys
     ↓              ↓                    ↓
  Plaintext    Encrypt/Decrypt      Key Storage
     ↓              ↓                    ↓
  Ciphertext   Audit Logged       Auto-Rotated
```

## Key Operationen

### Verschlüssele Evidence

```bash
vault write transit/encrypt/ssid-evidence \
  plaintext=$(base64 <<< '{"run_id": "20260314", "hash": "abc123"}')
```

### Entschlüssele Evidence

```bash
vault write transit/decrypt/ssid-evidence \
  ciphertext="vault:v1:8SDd3WHDOjf7mq69..."
```

### Signiere Hash Chain Block

```bash
vault write transit/sign/ssid-chain \
  input=$(base64 <<< "$BLOCK_HASH")
```

## Policy (HCL)

```hcl
# ssid-ems-policy.hcl
path "transit/encrypt/ssid-evidence" {
  capabilities = ["update"]
}

path "transit/decrypt/ssid-evidence" {
  capabilities = ["update"]
}

path "transit/sign/ssid-chain" {
  capabilities = ["update"]
}

path "transit/verify/ssid-chain" {
  capabilities = ["update"]
}

# Deny Key Export — Applikationen sehen niemals rohes Keys
path "transit/export/*" {
  capabilities = ["deny"]
}
```

## Key Rotation

```bash
# Rotiere Key (automatisch, Zero-Downtime)
vault write -f transit/keys/ssid-evidence/rotate

# Setze Minimum Decryption Version (nach Migration)
vault write transit/keys/ssid-evidence min_decryption_version=5
```

## Integration mit Evidence System

Die Evidence Chain nutzt Vault Transit für:
1. **Block Signing** — jeder SEALED Block erhält eine Vault Signatur
2. **Content Encryption** — empfindliche Evidence Payload verschlüsselt at rest
3. **Key Rotation** — automatisierte Monats-Rotation mit Audit Trail
