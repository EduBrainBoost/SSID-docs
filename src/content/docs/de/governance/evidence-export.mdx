# Public Evidence Export

SSID verwaltet eine **öffentliche Evidence Export** aller Governance Entscheidungen, Policy Gates und kritischen System-Operationen. Dieses Ledger ist unveränderlich (WORM - Write-Once-Read-Many) und kryptographisch verifizierbar.

## Was ist Evidence?

**Evidence** ist das unveränderliche Audit Trail jeder signifikanten Operation in SSID:

- **Policy Gate Ergebnisse:** Jede Gate Ausführung (Secret-Scan, Denylist-Gate, Structure-Guard, usw.)
- **Policy Änderungen:** Jedes Update zu OPA Regeln, Compliance Templates und Governance Policies
- **Smart Contract Deployments:** On-Chain Anker und Contract Deployment Records
- **Release Entscheidungen:** Jede Genehmigung, Merge und Release Gate
- **Incident Responses:** Incident Reports, Resolutionen und Post-Mortems
- **Compliance Audits:** Audit Läufe, Findings und Remediation Records

## Zugriff auf das Public Evidence Log

Die **Public Evidence Export** ist als sanitierter JSON/CSV Export verfügbar:

```bash
# Download Evidence Export
curl -X GET https://ssid-docs.github.io/evidence-export.json

# Oder via SSID-open-core
git clone https://github.com/EduBrainBoost/SSID-open-core.git
cat public-export/evidence-manifest.json
```

## Hash Chain Verifikation

Jeder Evidence Eintrag ist mit dem vorherigen über eine **Hash Chain** verlinkt. Sie können das gesamte Audit Trail verifizieren:

```bash

python verify-evidence-chain.py evidence-export.json

# Output:

# ✓ Entry 2: hash valid, linked to parent
# ...
# ✓ Chain integrity: VERIFIED
# Total entries: 1,234
```

## Verifikation von Evidence Authentizität

Alle Evidence sind signiert mit SSID's Root Keys:

```bash
# Verifiziere einen einzelnen Eintrag
gpg --verify evidence-entry-123.json.sig evidence-entry-123.json

# Batch Verifikation
openssl dgst -sha256 -verify public-key.pem \
  -signature evidence-entry-123.sig \
  evidence-entry-123.json

# Output: Verified OK
```

## Privacy & Sanitization

Die Öffentliche Evidence Export enthält **keine sensitiven Daten:**

✅ **Was wird exportiert:**
- Gate Namen und Ergebnisse (PASS/FAIL)
- Datei Counts, nicht Datei Namen
- Timestamp und Agent ID
- Hash Werte (für Verifikation, nicht Inhalt)
- Compliance Findings (redactiert)

❌ **Was wird NICHT exportiert:**
- PII (Namen, Emails, Adressen)
- Private Keys, Zugangsdaten, API Keys
- Interne IP Adressen oder Hostnames
- Vulnerability Details (vor Public Disclosure)
- Customer Daten oder Transaction IDs

- **IPFS:** Verteilte Kopie via IPFS für dezentralisierten Zugriff

## Fragen & Verifikation

Um spezifische Evidence zu verifizieren oder Evidence über einen bestimmten Incident anzufordern:

```bash
# Query Evidence nach Datums-Range
curl "https://ssid-docs.github.io/evidence-export.json?from=2026-01-01&to=2026-04-13"

# Query nach Gate Name
curl "https://ssid-docs.github.io/evidence-export.json?gate=secret-scan"

# Query nach Operation Type
curl "https://ssid-docs.github.io/evidence-export.json?operation=policy-update"
```
