# DSGVO-Konformitaet

SSID ist nach den Grundsaetzen **Privacy by Design** und **Privacy by Default** konzipiert — Kernanforderungen der EU-Datenschutz-Grundverordnung (DSGVO).

## Kernzuordnungen

| DSGVO-Grundsatz | SSID-Umsetzung |
|---------------|---------------------|
| **Datenminimierung** | Nur wesentliche Identitaetsattribute werden gespeichert |
| **Zweckbindung** | Nachweise definieren erlaubte Anwendungsfaelle |
| **Recht auf Loeschung** | Nutzer koennen Nachweise widerrufen und Wallet-Daten loeschen |
| **Recht auf Portabilitaet** | Standard-DID/VC-Formate ermoeglichen Interoperabilitaet |
| **Privacy by Design** | Architektur erzwingt Datenschutz von Grund auf |
| **Einwilligung** | Nachweis-Weitergabe erfordert explizite Nutzerzustimmung |

## Selbstbestimmung und DSGVO

SSI ist von Natur aus DSGVO-freundlich, weil:

1. **Der Nutzer ist Verantwortlicher** — er entscheidet, was geteilt wird
2. **Keine zentrale Datenbank** — personenbezogene Daten werden nicht aggregiert
3. **Selective Disclosure** — nur benoetigte Attribute teilen, nicht vollstaendige Nachweise
4. **Widerrufbarkeit** — Nachweise koennen jederzeit widerrufen werden

<Aside type="note">
SSID speichert keine personenbezogenen Daten on-chain oder in oeffentlichen Repositories. Alle personenbezogenen Daten verbleiben im Wallet des Nutzers unter dessen Kontrolle.
</Aside>

## Verarbeitungsregister

| Entity | Rolle | Verarbeitungsumfang |
|--------|-------|-------------------|
| **Nutzer** | Verantwortlicher | Besitzt DID, Wallet, Credential-Speicher |
| **SSID-Netzwerk** | Auftragsverarbeiter | Koordiniert Verifizierungsablaeufe, erteilt signierte Nachweise |
| **Evidence Layer** | Externer WORM-Speicher | Speichert Hashes + Compliance-Bescheinigungen (keine PII) |

**Konformitaetsmodell:** SSID fungiert als **Auftragsverarbeiter im Auftrag von Verantwortlichen** (Nutzern) und ist niemals selbst Verantwortlicher. Nutzer behalten volle Kontrolle ueber ihre Identitaetsdaten.

## Verantwortlicher vs. Auftragsverarbeiter

### Nutzer als Verantwortlicher
- **Verantwortung:** Legale Grundlage fuer die Ausstellung von Nachweisen, Nutzerzustimmung
- **Rechte:** Recht auf Zugriff, Berichtigung, Loeschung eigener Identitaetsdaten
- **Durchsetzung:** Verwalten ueber lokale DID-Wallet; SSID bietet keine zentrale Verwaltung

### SSID als Auftragsverarbeiter
- **Verantwortung:** Technische Schutzmaßnahmen (Verschluesselung in Transit/Ruhe, Zugriffskontrolle)
- **Verarbeitungsaktivitaeten:**
  - Akzeptieren von Nachweis-Uebermittlungen von Ausstellern
  - Ueberpruefen der Aussteller-Autorisierung (Policy Gates)
  - Koordinierung von Verifizierungsablaeufen (ZTA-Ablauf)
  - Generierung signierter Nachweise (Evidence Chain)
  - **NIEMALS personenbezogene Daten speichern; nur Hashes und Audit-Logs**
- **Datenverarbeitungsvereinbarung (DPA):** Standardbedingungen gelten; Nutzer akzeptieren ueber Nutzungsbedingungen
