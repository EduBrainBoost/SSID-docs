# Testnet Deployment Guide

Dieser Guide behandelt die Bereitstellung von SSID-Verträgen, -Diensten und -Identity-Flows auf **öffentlichen Testnets** für Entwicklung und Integrationstests vor dem Mainnet.

## Unterstützte Testnets

### Ethereum Sepolia

- **Network ID:** 11155111
- **RPC Endpoint:** https://sepolia.infura.io/v3/YOUR_INFURA_KEY
- **Block Explorer:** https://sepolia.etherscan.io
- **Faucet:** https://sepoliafaucet.com (erfordert Alchemy-Anmeldung) oder https://www.alchemy.com/faucets/ethereum-sepolia
- **Native Token:** Sepolia ETH (kostenlos, vom Faucet anfordern)
- **Chain Duration:** Permanentes Testnet (kein Reset-Zeitplan)

**Verwenden für:**
- Smart-Contract-Tests (Identity-Verträge, Governance-Tokens)
- KYC/KYB-Flow-Validierung
- WORM-Ledger-Anker-Validierung (optional)

### Polygon Mumbai

- **Network ID:** 80001
- **RPC Endpoint:** https://rpc-mumbai.maticvigil.com oder https://mumbai.infura.io/v3/YOUR_INFURA_KEY
- **Block Explorer:** https://mumbai.polygonscan.com
- **Faucet:** https://faucet.polygon.technology/ (erfordert Twitter/GitHub)
- **Native Token:** Polygon Mumbai MATIC (kostenlos, vom Faucet anfordern)
- **Chain Duration:** Permanentes Testnet (kein Reset-Zeitplan)

**Verwenden für:**
- Multi-Chain Identity Testing
- Cross-Chain DID Resolution (Bridge-Tests)
- VC-Ausstellung über EVM-Chains

## Voraussetzungen

1. **Wallet:** MetaMask oder äquivalent (Test-Konto mit Private Key sicher in Vault gespeichert)
2. **Testnet Funds:** Vom Faucet anfordern (Sepolia ETH, Mumbai MATIC — kostenlos, kein Mainnet-Risiko)
3. **RPC Credentials:** Infura, Alchemy oder öffentlicher Node (in `.env`, nie committed)
4. **Git:** Clone SSID-open-core für Contract ABIs und Deployment-Skripte

## Umgebungs-Setup

### 1. Abhängigkeiten installieren

```bash
cd SSID-orchestrator  # oder Deployment-Dienst
npm install hardhat @nomicfoundation/hardhat-toolbox ethers
pnpm install --frozen-lockfile
```

### 2. Testnet RPC konfigurieren

Erstellen Sie `.env.testnet` (nie committen):

```bash
# Sepolia
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/YOUR_INFURA_KEY
SEPOLIA_PRIVATE_KEY=0x...  # Nur Test-Konto, keine Mainnet-Schlüssel

# Mumbai
MUMBAI_RPC_URL=https://mumbai.infura.io/v3/YOUR_INFURA_KEY
MUMBAI_PRIVATE_KEY=0x...   # Nur Test-Konto, keine Mainnet-Schlüssel

# Vault (Produktionsreferenz)
VAULT_ADDR=https://vault.ssid.dev  # Staging, nicht Prod
VAULT_TOKEN=hvs_...  # Nur Staging-Token
```

### 3. Testnet Funds anfordern

**Sepolia ETH:**
```bash
# Besuchen Sie https://sepoliafaucet.com
# Geben Sie Ihre Wallet-Adresse ein
# Erhalten Sie 0,5 ETH (täglich wiederholbar)
# Überprüfen Sie im Block Explorer: https://sepolia.etherscan.io/address/YOUR_ADDRESS
```

**Mumbai MATIC:**
```bash
# Besuchen Sie https://faucet.polygon.technology/
# Wählen Sie Mumbai (Polygon Testnet)
# Geben Sie Ihre Wallet-Adresse ein
# Erhalten Sie 0,5 MATIC (alle 24 Stunden wiederholbar)
# Überprüfen Sie: https://mumbai.polygonscan.com/address/YOUR_ADDRESS
```

## Deploy Identity Contracts

### Sepolia Deployment

```bash
# 1. Compile Verträge
npx hardhat compile --network sepolia

# 2. Deploy Identity Resolver
npx hardhat run scripts/deploy-identity.js --network sepolia

# 3. Verify im Block Explorer
# (Contract-Adresse wird in stdout gedruckt)
# Besuchen Sie: https://sepolia.etherscan.io/address/0x...
```

**Erwartete Ausgabe:**
```
IdentityResolver deployed to: 0x1234567890abcdef...
  Network: Sepolia (11155111)
  Transaction: 0xabcd...
  Gas Used: ~1.2M
```

### Mumbai Deployment

```bash
# 1. Deploy auf Mumbai
npx hardhat run scripts/deploy-identity.js --network mumbai

# 2. Verify
# Besuchen Sie: https://mumbai.polygonscan.com/address/0x...
```

## Register DIDs auf Testnet

Nach der Bereitstellung, registrieren Sie Identity-Verträge auf der Chain:

```bash
# 1. Generieren Sie Test-DID
node scripts/generate-test-did.js --network sepolia

# 2. Anchor DID zu Sepolia
npx hardhat run scripts/anchor-did.js --network sepolia \
  --did did:ssid:sepolia:0x1234567890abcdef... \
  --resolver 0xresolver... \
  --key-material-hash sha3(...) \
  --did-document ./test-did.json

# 3. Verify DID-Auflösung
curl -X GET http://localhost:3310/did-resolution \
  -H "Content-Type: application/json" \
  -d '{"did":"did:ssid:sepolia:0x1234567890abcdef..."}'

# Erwartet: 200 OK mit DID-Dokument, Public Key Material
```

## Integration Tests ausführen

### Full Flow Test (Sepolia + Mumbai)

```bash
# 1. Umgebung setzen
export TESTNET=sepolia,mumbai
export RPC_SEPOLIA=https://sepolia.infura.io/v3/YOUR_KEY
export RPC_MUMBAI=https://mumbai.infura.io/v3/YOUR_KEY

# 2. Führen Sie vollständigen Identity Flow aus
pnpm test:integration --testnet

# 3. Erwartete Ergebnisse:
#   ✓ DID-Generierung
#   ✓ Sepolia Contract Deploy
#   ✓ Mumbai Cross-Chain Anchor
#   ✓ VC-Ausstellung
#   ✓ Verifizierung
```

## Testnet Limitierungen & Notizen

⚠️ **Keine Garantien:**
- Der Testnet-Status kann ohne Ankündigung zurückgesetzt werden (historisch selten für Sepolia/Mumbai)
- Transaction Finality kann langsamer sein als auf Mainnet
- Gas-Schätzung kann variieren

⚠️ **Nicht für Produktion:**
- Alle Testnet-Mittel sind wertlos (kein Mainnet-Äquivalent)
- Private Keys sollten nur Test-Keys sein, niemals Mainnet-Keys wiederverwenden
- Keine SLA oder Garantie für Verfügbarkeit

✅ **Testing Best Practices:**
- Testen Sie immer vollständige Flows End-to-End (DID → VC → Verifizierung)
- Verifizieren Sie Verträge auf Block Explorer nach dem Deploy
- Verwenden Sie Vault Staging für Credential-Tests (niemals Prod-Credentials)
- Halten Sie Testnet-Deployments separat von Mainnet in der Konfiguration

## Fehlerbehebung

**Contract Deploy fehlgeschlagen: "Insufficient gas"**
```bash
# Erhöhen Sie das Gas-Limit
npx hardhat run scripts/deploy-identity.js --network sepolia \
  --gas-limit 2000000
```

**DID Resolution gibt 404 zurück**
```bash
# Verifizieren Sie, dass der Contract bereitgestellt ist
curl https://sepolia.etherscan.io/api?module=account&action=getcode&address=0x...

# Wenn Code leer ist, hat der Contract nicht deployed
# Führen Sie Deployment erneut aus und überprüfen Sie den Transaction Status
```

**Testnet Faucet Rate Limited**
```bash
# Warten Sie 24 Stunden zwischen Anforderungen, oder
# Verwenden Sie mehrere Wallet-Adressen zum Testen
# Oder fordern Sie Funds in Project Discord/Community Channels an
```
