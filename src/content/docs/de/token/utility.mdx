# Token Utility & Governance

Das SSID Token ist ein **Utility Token**, das Partizipation im SSID Ökosystem ermöglicht.

<Aside type="caution">
Das SSID Token ist KEIN Investitionsprodukt, Zahlungsinstrument oder Security. Es bietet funktionalen Zugang zu SSID Governance und Services.
</Aside>

## Token Funktionen

### Governance
- **Proposal Voting** — Token Holder können über Governance Proposals abstimmen
- **Parameter Adjustment** — Abstimmung über System Parameter
- **Policy Gate Modifizierungen** — Gate-Änderungen vorschlagen und genehmigen

### Utility
- **Staking** — Tokens staken um Validator Nodes zu betreiben
- **Service Zugang** — Bestimmte Premium Services erfordern Token Staking
- **Reward Distribution** — Contributors verdienen Tokens für Ökosystem-Beiträge

### Incentive Alignment
- **Bug Bounties** — Rewards für verantwortungsvolle Vulnerability Disclosure
- **Documentation Contributions** — Rewards für Verbesserung öffentlicher Dokumentation
- **Code Contributions** — Rewards für akzeptierte PRs zu Open-Core

## Token Eigenschaften

| Eigenschaft | Wert |
|-------------|------|
| **Typ** | Utility / Governance |
| **Zahlung erforderlich?** | Nein — Core Identity Features sind kostenlos |
| **Custody** | Non-Custodial (User hält Keys) |
| **Transfer** | Peer-to-Peer, kein Intermediary erforderlich |
| **Inflation** | Von Community Vote governiert |

## Keine Zahlungsverpflichtung

<Aside type="note">
Core SSID Funktionalität (DID Creation, Credential Issuance, Verification) erfordert keine Token Ownership oder Zahlung. Das Token ist für Governance Partizipation und optionale Premium Features.
</Aside>
