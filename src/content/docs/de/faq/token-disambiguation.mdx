# Token Disambiguation

<Aside type="danger">
**SSID (Self-Sovereign Identity) ist NICHT angegliedert, unterstützt oder in irgendeiner Weise verbunden mit irgendwelchen Token, die auf Binance Smart Chain (BSC) oder einer anderen Exchange unter dem Namen "SSID Token" gehandelt werden.**
</Aside>

## Was ist offiziell?

| Offiziell | Nicht offiziell |
|----------|-------------|
| Diese Dokumentations-Site | Irgendwelche Token auf BSC/DEX mit "SSID" Ticker |
| [SSID-open-core](https://github.com/EduBrainBoost/SSID-open-core) Repository | Irgendwelche Social-Media-Konten, die nicht von offiziellen Repos verlinkt sind |
| SSID Utility Token (falls/wenn durch offizielle Governance gestartet) | Irgendwelche Airdrop, Presale oder Investitionsschema mit "SSID" Namen |

## Wie Sie überprüfen können

1. **Überprüfen Sie die Quelle** — Offizielle Informationen stammen von den oben aufgeführten GitHub-Repositories
2. **Überprüfen Sie Governance-Records** — Jeder offizielle Token-Start hat Evidence im WORM-Audit-Log
3. **Keine Presale** — SSID führt nicht durch und wird niemals eine Presale oder ICO durchführen
4. **Keine Rendite-Versprechen** — Der SSID-Token ist nur Utility

## Warum die Namen-Kollision?

"SSID" ist ein häufiges Akronym (wird auch für WiFi-Netzwerknamen verwendet). Namen-Kollisionen im Crypto-Raum sind häufig und oft absichtlich von Scam-Projekten. Diese Disambiguierungsseite existiert zum Schutz der Benutzer.

## Betrug melden

Wenn Sie auf ein Projekt treffen, das fälschlicherweise behauptet, mit SSID verbunden zu sein:

1. Interagieren Sie nicht damit
2. Melden Sie es auf der relevanten Exchange/Plattform
3. Benachrichtigen Sie das SSID-Team durch offizielle GitHub-Kanäle

<Aside type="caution">
Verifizieren Sie immer über offizielle Kanäle, bevor Sie mit irgendwelchen Token oder Projekten interagieren, die behaupten, SSID zu sein.
</Aside>
