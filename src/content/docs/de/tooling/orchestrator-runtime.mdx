# Orchestrator Runtime

Der **SSID Orchestrator** ist die Execution-Engine, die Tasks dispatcht, Workflows managed und isolierte Execution-Sessions maintained. Es verbindet die Control Plane (EMS) mit der Data Plane (SSID Core).

│  dispatch create --workflow identity...  │

## Kern-Konzepte

### Dispatcher

Der **Dispatcher** ist der Task-Router, der:
- Tasks von EMS empfaengt
- Zu verfuegbaren Workers routed
- Task-Queue und Prioritaeten managed
- Retries und Circuit Breaker handhabt

**Routing-Regeln:**
- **Prioritaet:** High → Normal → Low (FIFO pro Level)
- **Affinity:** Einige Tasks erfordern spezifische Worker (z.B. KYC Tasks mit Provider API Keys)
- **Timeout:** Default 3600s, configurable pro Task
- **Retry:** Exponentieller Backoff (1s, 2s, 4s, 8s, max 60s)

### Workflows

Ein **Workflow** ist ein Multi-Step-Prozess mit Branching und Parallelisierung.

**Built-in Workflows:**

| Workflow | Steps | Zweck | Beispiel |
|----------|-------|-------|---------|
| `identity-verification` | 3–5 | Erstelle und verifiziere Identitaet | KYC Flow |
| `credential-issuance` | 2–3 | Issue verifiable Credential | VC-Ausstellung |
| `credential-revocation` | 1–2 | Revoke ausgestelltes Credential | Credential-Lifecycle |
| `did-registration` | 1–2 | Registriere DID On-Chain | DID-Setup |
| `policy-audit` | 1 | Audit Compliance-Policies | Governance |

**Workflow DSL (YAML Beispiel):**

```yaml
name: identity-verification
steps:
  - id: validate-input
    type: validation
    timeout: 30s
    next: [check-kyc-provider]
    
  - id: check-kyc-provider
    type: external-api
    provider: sumsub  # oder iDenfy, Onfido, etc.
    timeout: 300s
    retry: 3
    next: [issue-credential, on-failure: manual-review]
    
  - id: issue-credential
    type: ssid-contract
    contract: VerifiableCredential
    timeout: 120s
    next: [complete]
    
  - id: manual-review
    type: human
    escalate: true
    timeout: 86400s
    
  - id: complete
    type: end-state
    outputs: [credential-id, verification-status]
```

### Sessions

Eine **Session** ist ein isolierter Execution-Context fuer einen Task mit:
- Eindeutige Session-ID
- Scoped Umgebungsvariablen
- Secrets Management (via Vault)
- State Persistence
- Timeout und Cleanup Policies

**Session-Lifecycle:**

```
CREATE (allocate resources)
  ↓
READY (environment set up)
  ↓
RUNNING (executing steps)
  ↓
CLEANUP (remove session state)
  ↓
CLOSED (resources released)
```

**Session-Isolation:**

```bash
session-123
├── .env (scoped variables)
├── .secrets/ (Vault-mounted)
├── state.json (execution state)
├── logs/ (stderr, stdout)
└── output/ (results)
```

#### Health Check

```bash
GET /health

  "version": "2.1.0",
  "workers": 4,
  "queue": {
    "pending": 2,
    "running": 3,
    "failed": 0
  }

```

#### Create Task

```bash
POST /dispatch/tasks
Content-Type: application/json

Request:
{
  "workflow": "identity-verification",
  "priority": "high",
  "timeout": 3600,
  "input": {
    "user_id": "user-123",
    "kyc_data": {...}
  },
  "callbackUrl": "https://example.com/webhook"  # optional

Response (201 Created):
{
  "taskId": "task-2026-04-13-001",
  "workflow": "identity-verification",
  "status": "PENDING",
  "createdAt": "2026-04-13T14:22:45Z",
  "estimatedDuration": "45s"
}
```

## Web UI

**URL:** `http://localhost:5273` (Entwicklung)

### Dashboard

Echtzeit-Ueberblick der Orchestrator-Gesundheit:

- **Workers:** Anzahl und Status (healthy/degraded/down)
- **Queue:** Pending Tasks nach Prioritaet
- **Throughput:** Tasks/min, Durchschnittsdauer, Error Rate (P95)
- **Alerts:** Kritische Probleme und Empfehlungen

### Task Monitor

Browse aktive und abgeschlossene Tasks:

- **Task List:** Filter nach Workflow, Status, Datumsbereich
- **Task Details:** Vollstaendige Task-Metadaten, Inputs, Outputs, Logs
- **Live Logs:** Stream Task-Logs in Echtzeit
- **Retry:** Manually Retry fehlgeschlagener Tasks
- **Cancel:** Stoppe laufende Tasks

### Workflow Viewer

Explore Workflow-Definitionen:

- **Workflow Graph:** Visual DAG von Steps
- **Step Details:** Beschreibung, Timeout, Retry-Policy
- **Example Runs:** Vergangene Executions mit Metriken
- **Metrics:** Durchschnittsdauer, Success Rate, Fehlertypen

## Integration Beispiel

### Complete Identity Verification Flow

```bash
# 1. Create task via API
curl -X POST http://localhost:3310/dispatch/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "workflow": "identity-verification",
    "priority": "high",
    "input": {
      "user_id": "user-123",
      "kyc_data": {
        "first_name": "Alice",
        "last_name": "Smith",
        "email": "alice@example.com",
        "country": "DE"
      }
    }
  }'

# Response: { "taskId": "task-001", "status": "PENDING" }

# 2. Monitor progress
curl -X GET http://localhost:3310/dispatch/tasks/task-001

# Response:
# {
#   "status": "RUNNING",
#   "progress": { "step": "check-kyc-provider", "percentage": 50 }
# }

# 3. Wait for completion (polling or webhook)
# When complete, receive:
# {
#   "status": "COMPLETED",
#   "output": {
#     "credential_id": "cred-abc123def",
#     "verification_status": "verified",
#     "vc_json": {...}
#   }
# }

# 4. Access the issued credential
curl -X GET http://localhost:3310/credentials/cred-abc123def
# Returns: Full verifiable credential (signed, on-chain anchored)
```

## Konfiguration

### Umgebungsvariablen

```bash
# Dispatcher
ORCHESTRATOR_WORKERS=4           # Number of worker processes
ORCHESTRATOR_QUEUE_SIZE=1000     # Max pending tasks
ORCHESTRATOR_TIMEOUT_DEFAULT=3600 # Default task timeout (seconds)

# Workflows
ORCHESTRATOR_WORKFLOW_PATH=/app/workflows  # Workflow definitions
ORCHESTRATOR_PLUGIN_PATH=/app/plugins      # Custom step plugins

# External Services
KYC_PROVIDER=sumsub              # Identity verification provider
KYC_API_KEY=... (in Vault)      # KYC provider credentials
SSID_CORE_URL=http://localhost:8080  # SSID Core service

# Sessions
SESSION_ISOLATION=true           # Enable session isolation
SESSION_TTL=7200                 # Session lifetime (seconds)
SESSION_CLEANUP_INTERVAL=300     # Cleanup check interval (seconds)

# Web UI
ORCHESTRATOR_WEB_PORT=5273      # UI server port
ORCHESTRATOR_API_PORT=3310      # API server port
```

### Vault Integration

Alle sensitiven Daten (API Keys, Private Keys) werden in Vault gespeichert:

```bash
# Orchestrator hat Vault Token scoped zu:
# secret/orchestrator/kyc-provider
# secret/orchestrator/ssid-signer
# secret/data/vc-private-key

# Keine Secrets im Code oder .env Dateien
```

## Troubleshooting

**Task Stuck in PENDING**

```bash
# Check worker health
curl http://localhost:3310/health

# If workers=0, restart orchestrator:
pm2 restart orchestrator

# If workers>0, task may be waiting for resources:
curl http://localhost:3310/dispatch/tasks/task-id/logs | head -20
```

**External Service Timeout (KYC Provider)**

```bash
# KYC provider slow; increase timeout:
POST /dispatch/tasks
{
  "workflow": "identity-verification",
  "timeout": 600  # increase from 300s to 600s
}
```

**Session Isolation Error**

```bash
# Session couldn't isolate; check Vault connectivity:
curl http://localhost:3310/health

# Verify Vault token:
echo $VAULT_TOKEN | base64 -d

# Re-authenticate if needed:
vault login -method=approle \
  role_id=$VAULT_ROLE_ID \
  secret_id=$VAULT_SECRET_ID
```
