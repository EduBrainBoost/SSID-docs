# MiCA Positioning

The **Markets in Crypto-Assets (MiCA)** regulation establishes a comprehensive framework for crypto-assets in the EU. SSID's token design is informed by MiCA classification requirements.

## SSID Token Classification

<Aside type="caution">
The SSID token is a **utility token** — it provides access to system functionality, not investment returns. This positioning is intentional and designed to fall under MiCA's utility token provisions.
</Aside>

| MiCA Category | SSID Applicability |
|---------------|-------------------|
| **E-Money Token (EMT)** | Not applicable — SSID token is not pegged to fiat |
| **Asset-Referenced Token (ART)** | Not applicable — SSID token references no external assets |
| **Utility Token** | Applicable — provides access to governance and system services |

## Utility Token Properties

The SSID token is designed to:

1. **Grant governance participation** — voting on proposals
2. **Enable service access** — staking for validator roles
3. **Reward contributions** — incentivize ecosystem participation
4. **NOT function as payment** — no payment obligation acceptance

## MiCA Compliance Approach

| Requirement | SSID Response |
|-------------|---------------|
| **White Paper** | Public documentation of token mechanics (this site) |
| **Consumer Protection** | Non-custodial design, user controls keys |
| **Transparency** | Open-core codebase, auditable governance |
| **No Misleading Claims** | Clear disclaimer: not an investment product |
