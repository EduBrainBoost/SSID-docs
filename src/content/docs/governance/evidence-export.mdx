# Public Evidence Export

SSID maintains a **public evidence export** of all governance decisions, policy gates, and critical system operations. This ledger is immutable (WORM - Write-Once-Read-Many) and cryptographically verifiable.

## What is Evidence?

**Evidence** is the immutable audit trail of every significant operation in SSID:

- **Policy Gate Results:** Every gate execution (secret-scan, denylist-gate, structure-guard, etc.)
- **Policy Changes:** Every update to OPA rules, compliance templates, and governance policies
- **Smart Contract Deployments:** On-chain anchors and contract deployment records
- **Release Decisions:** Every approval, merge, and release gate
- **Incident Responses:** Incident reports, resolutions, and post-mortems
- **Compliance Audits:** Audit runs, findings, and remediation records

## Accessing the Public Evidence Log

The **public evidence export** is available as a sanitized JSON/CSV export:

```bash
# Download evidence export
curl -X GET https://ssid-docs.github.io/evidence-export.json

# Or via SSID-open-core
git clone https://github.com/EduBrainBoost/SSID-open-core.git
cat public-export/evidence-manifest.json
```

## Hash Chain Verification

Every evidence entry is linked to the previous one via a **hash chain**. You can verify the entire audit trail:

```bash
# Verify hash chain integrity
python verify-evidence-chain.py evidence-export.json

# Output:
# ✓ Entry 1: hash valid, linked to parent
# ✓ Entry 2: hash valid, linked to parent
# ...
# ✓ Chain integrity: VERIFIED
# Total entries: 1,234
```

## Verifying Evidence Authenticity

All evidence is signed using SSID's root keys:

```bash
# Verify a single entry
gpg --verify evidence-entry-123.json.sig evidence-entry-123.json

# Batch verification
openssl dgst -sha256 -verify public-key.pem \
  -signature evidence-entry-123.sig \
  evidence-entry-123.json

# Output: Verified OK
```

## Privacy & Sanitization

The public evidence export contains **no sensitive data:**

✅ **What is exported:**
- Gate names and results (PASS/FAIL)
- File counts, not file names
- Timestamp and agent ID
- Hash values (for verification, not content)
- Compliance findings (redacted)

❌ **What is NOT exported:**
- PII (names, emails, addresses)
- Private keys, credentials, API keys
- Internal IP addresses or hostnames
- Vulnerability details (before public disclosure)
- Customer data or transaction IDs
