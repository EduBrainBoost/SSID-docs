## Overview

[Vault Transit](https://developer.hashicorp.com/vault/docs/secrets/transit) provides encryption as a service — applications never see raw key material.

## Architecture

```
Application → Vault Transit API → HSM-backed keys
     ↓              ↓                    ↓
  Plaintext    Encrypt/Decrypt      Key Storage
     ↓              ↓                    ↓
  Ciphertext   Audit Logged       Auto-Rotated
```

## Key Operations

### Encrypt Evidence

```bash
vault write transit/encrypt/ssid-evidence \
  plaintext=$(base64 <<< '{"run_id": "20260314", "hash": "abc123"}')
```

### Decrypt Evidence

```bash
vault write transit/decrypt/ssid-evidence \
  ciphertext="vault:v1:8SDd3WHDOjf7mq69..."
```

### Sign Hash Chain Block

```bash
vault write transit/sign/ssid-chain \
  input=$(base64 <<< "$BLOCK_HASH")
```

## Policy (HCL)

```hcl
# ssid-ems-policy.hcl
path "transit/encrypt/ssid-evidence" {
  capabilities = ["update"]
}

path "transit/decrypt/ssid-evidence" {
  capabilities = ["update"]
}

path "transit/sign/ssid-chain" {
  capabilities = ["update"]
}

path "transit/verify/ssid-chain" {
  capabilities = ["update"]
}

# Deny key export — applications never see raw keys
path "transit/export/*" {
  capabilities = ["deny"]
}
```

## Key Rotation

```bash
# Rotate key (automatic, zero-downtime)
vault write -f transit/keys/ssid-evidence/rotate

# Set minimum decryption version (after migration)
vault write transit/keys/ssid-evidence min_decryption_version=5
```

## Integration with Evidence System

The evidence chain uses Vault Transit for:
1. **Block signing** — each SEALED block gets a Vault signature
2. **Content encryption** — sensitive evidence payload encrypted at rest
3. **Key rotation** — automated monthly rotation with audit trail
