# Shards & Hybrid Charts

## What are Shards?

Shards are **cross-cutting architectural concerns** that apply uniformly across all 24 roots. They ensure consistent structure and enforce separation of concerns.

## Shard Anatomy

Each shard within a root follows a predictable file structure:

```
{root_number}_{root_name}/{shard_name}/
├── index.ts          # Public API
├── {shard}.impl.ts   # Implementation
├── {shard}.test.ts   # Tests
├── {shard}.schema.ts # Validation schema
└── manifest.json     # Hybrid chart entry
```

## Hybrid Charts

A hybrid chart is a **deterministic manifest** that binds a Root×Shard intersection to its:

1. **Content hash** — SHA-256 of all source files in the intersection
2. **Policy gates** — Required reviews/checks before modification
3. **Dependency graph** — Other intersections this one depends on
4. **Evidence pointers** — Links to audit log entries

<Aside type="note">
Hybrid charts are auto-generated by the SSID dispatcher. Manual editing triggers a policy violation.
</Aside>

## Deterministic Properties

| Property | Guarantee |
|----------|-----------|
| **Reproducibility** | Same inputs always produce same artifact hash |
| **Traceability** | Every change links to a PR, evidence entry, and policy gate check |
| **Isolation** | Shards within a root can be tested independently |
| **Composability** | Any Root×Shard can declare dependencies on other intersections |

## Example: Audit Logging × Repositories

```json
{
  "root": "02_audit_logging",
  "shard": "repositories",
  "version": "1.0.0",
  "artifact_hash": "sha256:e3b0c44298fc1c149afbf4c8996fb924...",
  "policy_gates": [
    "pr_review_required",
    "evidence_append_only",
    "no_direct_commit"
  ],
  "dependencies": [
    "03_crypto_primitives/services",
    "09_storage/adapters"
  ],
  "evidence_ref": "02_audit_logging/evidence/shard_repos/latest.json"
}
```
