# Token Utility & Governance

The SSID token is a **utility token** that enables participation in the SSID ecosystem.

<Aside type="caution">
The SSID token is NOT an investment product, payment instrument, or security. It provides functional access to SSID governance and services.
</Aside>

## Token Functions

### Governance
- **Proposal voting** — Token holders can vote on governance proposals
- **Parameter adjustment** — Vote on system parameters
- **Policy gate modifications** — Propose and approve gate changes

### Utility
- **Staking** — Stake tokens to operate validator nodes
- **Service access** — Certain premium services require token staking
- **Reward distribution** — Contributors earn tokens for ecosystem contributions

### Incentive Alignment
- **Bug bounties** — Rewards for responsible vulnerability disclosure
- **Documentation contributions** — Rewards for improving public docs
- **Code contributions** — Rewards for accepted PRs to open-core

## Token Properties

| Property | Value |
|----------|-------|
| **Type** | Utility / Governance |
| **Payment required?** | No — core identity features are free |
| **Custody** | Non-custodial (user holds keys) |
| **Transfer** | Peer-to-peer, no intermediary required |
| **Inflation** | Governed by community vote |

## No Payment Obligation

<Aside type="note">
Core SSID functionality (DID creation, credential issuance, verification) does not require token ownership or payment. The token is for governance participation and optional premium features.
</Aside>
