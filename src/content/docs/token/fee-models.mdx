# Fee Models

SSID implements a transparent, non-custodial fee model that routes protocol fees and developer rewards through deterministic rules. Commercial overlays such as enterprise subscriptions may exist, but they must not violate the non-custodial or non-profit constraints of the base protocol.

<Aside type="caution">
SSID fees are not profit, yield, or APR. They are service fees routed by protocol rules. No party receives a guaranteed financial return.
</Aside>

## Protocol Fee Distribution

All on-chain transactions that interact with SSID identity services are subject to a total fee of **3%**, split as follows:

| Component | Rate | Destination |
|-----------|------|-------------|
| System Fee | 2% | Non-profit treasury (`treasury_nonprofit_only`) |
| Developer Reward | 1% | Contributing developer wallets |
| **Total** | **3%** | |

### System Fee

The system fee funds:

- Infrastructure operation costs
- Security audits and bug bounties
- Open-source maintenance
- Compliance and legal requirements

### Developer Reward

The developer reward is distributed to contributors whose code is deployed in the active system. Distribution is calculated from the `license_fee_matrix.yaml`, which maps contributions across the active roots.

<Aside type="note">
Every root that contains deployed production code can generate reward eligibility for its contributors.
</Aside>

## Fee Routing

The fee routing logic is designed to be:

- **Immutable** — Core percentages are protocol-governed, not operator-controlled
- **Non-custodial** — Fees route directly to destination wallets or protocol sinks
- **Autonomous** — No manual intervention is required for routine distribution
- **Deterministic** — Given the same input, the same routing outcome is produced

## Enterprise Overlays

Enterprise deployments can layer subscriptions, service packaging, and payout preferences on top of the protocol fee model, provided they remain consistent with SSID’s constraints:

- No custody or escrow implication
- No investment-style framing
- No guaranteed returns
- No override of protocol safety, governance, or compliance rules

Typical overlays may include:

- Subscription tiers for support or throughput
- Governance-approved payout preferences
- Post-quantum or compliance-focused service bundles

## What This Is Not

<Aside type="danger">
The SSID fee model is not:

- An investment return or yield mechanism
- A profit-sharing arrangement
- An APR/APY product
- A staking reward in the DeFi sense
</Aside>
